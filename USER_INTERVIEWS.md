USER INTERVIEWS

Interview 1 — A.K. (Frontend Developer at Seed-stage Startup)

Background

* Team size: 6
* Uses: ChatGPT Team, GitHub Copilot
* Primary workflows: coding, debugging, documentation

Key Quotes

> "We honestly don’t track AI spending carefully unless it spikes."

> "Different team members buy tools individually, so costs get fragmented."

> "I’d use a tool like this if it gave me actionable recommendations instead of generic budgeting advice."

Most Surprising Insight

The company was paying for overlapping AI tools without a clear policy on when each should be used.

What Changed in My Product

Initially, I only planned to show total savings. After this interview, I added per-tool recommendations with reasoning because users wanted specific actionable suggestions.

Interview 2 — R.S. (Indie Hacker / Solo Builder)

Background

* Solo developer
* Uses: Claude Pro, OpenAI API

Key Quotes

> "API bills are harder to understand than subscriptions."

> "I don’t care about saving $5 — I care about identifying waste."

> "If the report looked shareable, I’d probably post it on Twitter."

Most Surprising Insight

The user cared more about clarity and transparency than maximizing savings.

What Changed in My Product

I simplified the audit UI and focused more on explainable recommendations rather than aggressive optimization claims.

Interview 3 — P.M. (Students)

Background

* Team size: 3
* Uses: Cursor, ChatGPT Plus
* Primary workflows: coding and product writing

Key Quotes

> "We started stacking tools without reevaluating older subscriptions."

> "I’d rather see one strong recommendation than ten weak ones."

> "The trustworthiness of the audit matters more than fancy visuals."

Most Surprising Insight

The user immediately lost trust when savings estimates looked mathematically unrealistic.

What Changed in My Product

I refactored the audit engine so recommendations are no longer double-counted in total savings calculations.
